##### 좋은 소스코드의 기준
좋은 소스코드는 눈을 편하게 해야한다. (빈칸, 정렬, 코드의 순서)
- 코드를 읽는 사람이 이미 친숙한, 일관성 있는 레이아웃을 사용하라.
- 비슷한 코드는 서로 비슷해 보이게 만들어라.
- 서로 연관된 코드는 하나의 블록으로 묶어라.

##### 일관성과 간결성을 위해서 줄 바꿈을 재정렬하기
```JAVA
public class Test {
  // Test (키, 몸무게, 이름) 주석으로 매개변수 표시 후 처리
  public static final Test testMethod =
        new Test (175, 66, "김세환");
  
  // 위에 코드가 밑에 코드보다 보기가 편하다.
  public static final Test testMethod =
    new Test(
    175, // 키
    66,  // 몸무게
    "김세환", //이름
    );
}
```
##### 메소드를 활용하여 불규칙성을 정리하라
함수와 매개변수가 길어서 어떤 로직은 줄바꿈이 일어나고 어떤로직은 한줄에 처리되는 그런 상황이 발생하게 된다면
새로운 함수를 만들어서 매개변수만 처리하고 새로만든 함수 내에서 기능을 처리하여 코드를 보기 편하게 수정하는 것이 좋다.
- 중복된 코드를 제거해 코드가 더 간결해졌다.
- 이름이나 에러 문자열 같은 테스트의 중요 부분들이 한 눈에 보이게 모아졌다.
- 새로운 테스트 추가가 훨씬 쉬워졌다.

##### 주석의 목적
주석의 목적은 코드를 읽는 사람이 코드를 작성한 사람만큼 코드를 잘 이해하게 돕는 데 있다.

##### 불필요한 주석
코드를 빠르게 유추할 수 있는 내용은 주석으로 달지 말라 주석 읽는 시간만 낭비할 뿐이다.

##### 나쁜 이름에 주석을 통한 설명을 자제하고 이름을 고칠 생각을 해라.

##### 주석으로 나의 코드작성 이유를 정리해라
이 코드가 왜 이런 함수를 사용하였는지, 혹은 이런 함수밖에 사용하지 못했는지, 어떠한 부분에 막혀 이런 로직밖에 작성할 수 없었는지 등을 기록하면 다른 개발자가 관련 부분을 이해하기도 쉽고,
잘못된 판단으로 테스트케이스를 작성하여 테스트를 하는 등의 시간을 아낄 수 있다.

##### 주석을 통해 해야 할 일을 작성하기

- FIXME     : 오작동을 일으킨다고 알려진 코드
- HACK      : 아름답지 않은 해결책
- XXX       : 위험! 여기 큰 문제가 있다.
- TextMate  : ESC

##### 코드의 의도를 명시하라
정렬 함수를 사용할 때 아래와 같은 주석이 있다면 둘다 이해는 할 수 있지만 개발자의 의도를 명시한다면 혹여 잘못된 데이터가 반환되더라도
다른 개발자가 문제점을 파악하기 쉬워진다.
- 리스트를 역순으로 반복한다. (x)
- 각 가격을 높은 값에서 낮은 값 순으로 나타낸다 (o)

##### if/else 블록의 순서
- 부정이 아닌 긍정을 다루어라.
- 간단한 것을 먼저 처리하라
```JAVA
  if(a == b) { // 긍정을 선호
  
  } else {
  
  }
  //----------------------
  if(a != b) {

  }
```

##### 삼항연산자 사용조건
- 삼항 연산자는 간단한 조건문을 처리할 때 한줄로 작성하여 가독성을 높이기 위한 것이다.
- 복잡한 연산을 삼항연산자로 처리하게 된다면 가독성이 떨어져 if/else 보다 좋지 않다.

##### do while 문 사용을 지양해라
- 조건은 눈에 띄는곳에 미리 나타나도록 하는 것을 선호한다. do while문은 조건이 아래에 있기 때문에 에러와 혼동의 원인이 된다.

##### 중첩 최소화

##### 코드를 분리하라
하나의 함수 안에 해야할 일이 많을수록 코드의 가독성도 떨어지고 결합도 역시 높아진다. 한 가지 예로 데이터를 주입하는 기능과 데이터를 처리하는 기능이 하나의 함수에 존재한다면
결합도도 높아지고 하나의 함수를 이해하는 데 더 많은 시간이 소요된다. 데이터를 주입하는 함수를 분리하고 데이터를 처리하는 함수에서 데이터를 주입하는 함수를 호출하여 처리만 한다면 다른 사람이 코드를 이해하는데 훨씬 수월할 것이다.

##### 기능 별 위치 정렬
하나의 함수의 한가지의 기능만 존재하는 것이 코드의 재사용성을 높이고 결합도를 낮춰 독립성을 갖는 코드를 가질 것이다. 허나 위와 같은 사례로 자잘한 기능들을 함수별로 나눈다면 오히려 가독성이 떨어지는 상황도 발생하게 된다. 이럴 때 하나의 함수에 자잘한 여러개의 기능들이 들어갈 수 있는데, 이 때 함수 내에서 뒤죽박죽 섞에서 기능을 처리하지 말고 기능별로 위치를 잡고 처리한다면 하나의 함수에서도 충분히 가독성 좋은 코드를 완성 할 수 있다.
